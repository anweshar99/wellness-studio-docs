## 2. Writing Principles
- **Be task-oriented:** Focus on what the user wants to accomplish.  
- **Use short, clear sentences:** Prefer active voice.  
- **Show steps as numbered lists.**  
- **Use headings consistently:** Each topic begins with a clear `H1`, subtopics use `H2` or `H3`.  
- **Avoid product jargon** unless essential; define any necessary term once.  
- **Use bold for UI elements** and *italics* for emphasis.

## 4. Formatting Standards
- Use **sentence case** for headings and button text.  
- Time format: `09:00 AM` not `9 am`.  
- Date format: `DD MMM YYYY` (e.g., `10 Nov 2025`).  
- Write numbers as digits (`3 staff members`, not `three`).  
- Keep line length ~80 characters in Markdown for readability.  

---

## 5. Accessibility and Inclusivity
- Use gender-neutral terms (they/them).  
- Avoid idioms and cultural references that may confuse global users.  
- Ensure all images (if added later) have descriptive alt text.
